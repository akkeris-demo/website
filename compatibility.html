Akkeris vs Heroku

  • Both platforms focused on developer experience, rather than operations point of view.
  • Heroku has less flexibility in front-door, less flexibility in infrastructure networking.
  • Heroku has no on-premises installation, completely software as a service model.
  • In ability to add custom brokers or shared secrets via Vault©.
  • Heroku contains the most addons and services by far.
  • Akkeris is docker and CNCF focused, Heroku is buildpack (special sauce) focused, but does support Docker.
  • Heroku has limited abilities to integrate with private networks.
  • Heroku has a rich set of capabilities and covers development tools all the way to scaling.

Cloud Foundary vs Akkeris

  • Akkeris easily integrates into kubernetes and docker.
  • Cloud Foundary has less integrated support for log shuttling and log sessions.
  • Some of the commercial versions of Cloud Foundary can be fairly expensive and vary greatly in their capabilities.
  • Akkeris has the ability to route multiple apps behind a website, purchase certificates, order domains and more.
  • No native integrations with front-door (BigIP F5, Nginx, etc).
  • Akkeris is built to be more extensible with custom features and addon capabilities.
  • With both a rich UI and CLI toolset in addtion to focusing on a larger lifecycle of an engineers needs, Akkeris aims to provide not just a deployment system, but a full developer pluggable toolkit.
  • Cloud foundary is more mature and has been in market longer than almost any platform.

Akkeris vs Kubernetes

  • Akkeris sits on top of kubernetes to handle more aspects of an application life cycle than just its runtime.
  • Akkeris supports addons to allow engineers to request not just runtime, but services such as postgres, or mysql.
  • Support for routing and front-door http, domains, tls certificates allows anyone to immediately create a site and route it to any app.
  • Kuberentes is focused on application runtime; only one component of an application life cycle.
  • While akkeris sits on top of kuberentes it does not impose limitations on how other tools can integrate with it, allowing the best of both worlds.
